## 1. 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **긴장감 우선** | 모든 UI 요소가 긴장감 고조에 기여해야 한다 |
| **어둡고 강렬한 톤** | 다크 배경 + 빨강/금색 강조색으로 스릴 연출 |
| **심플한 조작** | 한 화면에 하나의 핵심 액션만 배치 |
| **모바일 퍼스트** | 터치 친화적 대형 버튼, 세로 레이아웃 기본 |
| **즉각적 피드백** | 모든 터치/클릭에 시각·청각 반응 |

### 5.2 핵심 컴포넌트 상세

#### ChamberDisplay (탄창 표시)

```
크기: 280 × 280px (모바일), 360 × 360px (데스크탑)
형태: 원형, 6개의 슬롯이 60° 간격으로 배치
색상:
  - 배경: Steel Gray (#2C3E50)
  - 미확인 슬롯: Dark (#333) + "?" 표시
  - 확인된 빈칸: Emerald 테두리 (#2ECC71) + "✅"
  - 현재 슬롯: Amber 하이라이트 (#F39C12) + 깜빡임
  - 꽝 슬롯: Blood Red (#E74C3C) + "💀"
회전: CSS transform: rotate() + transition
```

#### TriggerButton (방아쇠 버튼)

```
크기: 200 × 60px (모바일), 240 × 64px (데스크탑)
형태: rounded-full, 큰 터치 영역
색상:
  - 기본: Amber (#F39C12) 그라데이션
  - 호버: 밝아짐 + scale(1.05)
  - 클릭: 어두워짐 + scale(0.95)
  - 비활성: Gray (#666)
텍스트: "🔫 방아쇠 당기기" / Bold 18px
햅틱: 클릭 시 진동 50ms (모바일)
```

#### PlayerStatusRow (참가자 상태)

```
레이아웃: flex, 좌→우: 아이콘 | 이름 | 상태 | 턴 번호
상태별:
  - 현재 턴: 🎯 + Amber 배경 + 깜빡임
  - 대기: ⏳ + 기본 배경
  - 생존: ✅ + 초록 텍스트
  - 꽝: 💀 + 빨간 텍스트 + 취소선
  - 미진행: ── + 회색 텍스트
```

## 6. 인터랙션 설계

### 6.1 방아쇠 인터랙션 (핵심)

```
[버튼 표시]
    │
    ▼ (터치/클릭)
[버튼 누름 효과] ── scale(0.95), 어두워짐, 짧은 진동
    │
    ▼ (0.3초)
[연출 시작] ── 버튼 비활성화
    │
    ├── 찰칵 사운드 재생
    ├── 탄창 현재 칸 하이라이트 깜빡임
    │
    ▼ (0.5초 후 결과 공개)
    │
    ├── [생존] ──── 초록 플래시 + "찰칵!" + 체크 애니메이션
    │                 │
    │                 ▼ (1.5초 후)
    │              자동으로 다음 참가자 턴으로 전환
    │
    └── [꽝] ────── 빨간 플래시 + "💥 빵!" + 화면 흔들림 + 폭발
                      │
                      ▼ (2초 후)
                   결과 화면으로 전환
```

### 6.2 탄창 스핀 인터랙션

```
[장전 화면 진입]
    │
    ▼
[탄환 배치 연출] ── 탄환 아이콘이 탄창에 들어감 (0.8초)
    │
    ▼
[스핀 시작] ── 탄창 고속 회전 (rotate 0→3600°, 1.5초)
    │
    ▼
[감속] ── ease-out 커브로 점차 느려짐
    │
    ▼
[정지] ── 시작 위치 결정, 게임 화면으로 전환
```

### 6.3 심박 효과

```css
/* 기본 (낮은 긴장감) */
@keyframes heartbeat-slow {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.02); }
}
/* 주기: 1000ms (60 BPM) */

/* 높은 긴장감 */
@keyframes heartbeat-fast {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.05); }
}
/* 주기: 400ms (150 BPM) */
```

## 8. 접근성

### 8.1 키보드 네비게이션

| 키 | 동작 |
|----|------|
| `Tab` | 다음 인터랙티브 요소로 이동 |
| `Enter` / `Space` | 버튼 활성화 (방아쇠, 시작 등) |
| `Escape` | 모달/오버레이 닫기 |

### 8.2 ARIA 속성

```html
<!-- 탄창 -->
<div role="img" aria-label="리볼버 탄창. 6칸 중 2칸 확인됨. 현재 3번째 칸.">
  ...
</div>

<!-- 방아쇠 버튼 -->
<button aria-label="방아쇠 당기기. 현재 차례: 김 대리. 꽝 확률: 1/4">
  🔫 방아쇠 당기기
</button>

<!-- 생존 결과 -->
<div role="alert" aria-live="assertive">
  찰칵! 김 대리 생존!
</div>
```

### 8.3 포커스 관리
- 화면 전환 시 주요 액션 버튼에 자동 포커스
- 방아쇠 버튼은 연출 중 `disabled`, 완료 후 다시 포커스 이동
- 결과 화면에서 "다시 하기" 버튼에 자동 포커스

## 9. 사용자 흐름 (User Flow)

```
[사이트 접속]
     │
     ▼
[시작 화면]
     │
     ├── 참가자 이름 입력 (2~6명)
     ├── 게임 모드 선택
     ├── 벌칙 입력 (선택)
     │
     ▼
[게임 시작 클릭]
     │
     ▼
[장전 화면] ── 탄환 배치 + 스핀 연출 (2~3초)
     │
     ▼
[게임 화면]
     │
     ├── 참가자 A 턴 ── 방아쇠 클릭
     │   ├── 생존 → 다음 참가자
     │   └── 꽝 → 결과 화면
     │
     ├── 참가자 B 턴 ── 방아쇠 클릭
     │   ├── 생존 → 다음 참가자
     │   └── 꽝 → 결과 화면
     │
     └── ... (꽝 발생까지 반복)
          │
          ▼
     [결과 화면]
          │
          ├── [다시 하기] → 같은 설정으로 재시작 → [장전 화면]
          ├── [처음으로] → [시작 화면]
          └── [공유하기] → 클립보드 복사 / SNS 공유
```
