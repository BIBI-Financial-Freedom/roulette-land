## 1. 서비스 소개

**러시안 룰렛 — 꽝 뽑기**는 모임에서 벌칙 대상을 정할 때 사용하는 **웹 기반 파티 게임**이다.  
리볼버 탄창을 모티브로 한 6칸 회전식 UI에서 참가자들이 돌아가며 방아쇠를 당기고, 꽝(탄환)에 걸린 사람이 벌칙을 수행하는 심플하지만 긴장감 넘치는 게임이다.

### 1.1 한 줄 요약
> "누가 꽝에 걸릴까? — 러시안 룰렛으로 결정하는 벌칙 게임"

### 1.2 서비스 유형

| 항목 | 내용 |
|------|------|
| 서비스 형태 | 웹 애플리케이션 (SPA) |
| 접근 방식 | URL 접속 즉시 플레이 (설치 불필요) |
| 비용 | 무료 |
| 계정 | 불필요 (비로그인) |
| 데이터 저장 | 로컬 스토리지 (선택) |

## 2. 서비스 목표

### 2.1 단기 목표 (v1.0)
- 2~6명이 즉시 플레이 가능한 러시안 룰렛 게임 완성
- 클래식 모드 + 기본 연출(사운드, 애니메이션) 구현
- 모바일/데스크탑 반응형 지원
- GitHub Pages를 통한 무료 배포

### 2.2 중기 목표 (v2.0)
- 5가지 게임 모드 전체 구현 (클래식, 페어, 하드, 데스매치, 서바이벌)
- 벌칙 룰렛 연동 기능
- 결과 공유 기능 (카카오톡, 링크 복사)
- 역대 기록 저장 및 통계

### 2.3 장기 목표
- 실시간 멀티플레이 (원격 참가)
- 커스텀 테마 및 스킨
- 토너먼트/리그 모드

## 3. 타겟 사용자

### 3.1 주요 타겟

| 사용자 그룹 | 특성 | 사용 상황 |
|-------------|------|-----------|
| 직장인 | 점심/커피 시간 활용 | 커피내기, 점심값 결정 |
| 대학생 | 그룹 모임, 스터디 | 벌칙 게임, 술래 정하기 |
| 친구 모임 | 비공식 모임 | 파티 게임, 내기 |

### 3.2 사용자 페르소나

**페르소나 A — 김 대리 (30세, 직장인)**
- 매일 팀원 5명과 커피를 마시는데, 누가 살지 고르는 게 항상 어색함
- 공정하면서도 재미있는 방법을 원함
- 스마트폰으로 빠르게 접속해서 바로 플레이하고 싶음

**페르소나 B — 이 학생 (22세, 대학생)**
- 스터디 모임에서 발표 순서나 벌칙 대상을 정해야 함
- 단순 사다리타기는 재미없고, 좀 더 스릴 있는 방식을 원함
- 결과를 카카오톡으로 공유해서 증거(?)를 남기고 싶음

## 4. 사용 시나리오

### 시나리오 1: 커피내기 (기본)
```
1. 김 대리가 스마트폰으로 사이트에 접속
2. 팀원 5명의 이름 입력 (김, 이, 박, 최, 정)
3. 클래식 모드 선택, 벌칙: "아메리카노 5잔 사기"
4. 게임 시작 → 탄창 장전 & 스핀 연출
5. 김 대리부터 순서대로 방아쇠 클릭
   - 김: 찰칵! (생존) → 이: 찰칵! (생존) → 박: 💥 빵! (꽝!)
6. "박 대리님이 꽝! 아메리카노 5잔 사기!"
7. 결과 화면 캡처 → 단톡방 공유
```

### 시나리오 2: 서바이벌 (탈락전)
```
1. 친구 모임 6명이 접속
2. 서바이벌 모드 선택
3. 라운드 1: C가 꽝 → 탈락 (5명 남음)
4. 라운드 2: A가 꽝 → 탈락 (4명 남음)
5. ... 반복 ...
6. 최종 1명 생존 → 면제!
7. 탈락 순서대로 벌칙 등급 결정
```

### 시나리오 3: 하드 모드 (긴박한 결정)
```
1. 2명이 점심값 내기
2. 하드 모드 (탄환 2개) 선택
3. 첫 번째: 1/3 확률로 시작 → 극한 긴장감
4. 빠르게 꽝 발생 → 결과 확정
```

## 5. 서비스 핵심 가치

### 5.1 가치 피라미드

```
         ┌─────────────┐
         │   긴장감     │  ← 핵심 감정
         │  & 스릴      │
         ├─────────────┤
         │   공정성     │  ← RNG 기반 확률
         ├─────────────┤
         │   간편함     │  ← 설치 없이 즉시 플레이
         ├─────────────┤
         │   재미       │  ← 연출, 사운드, 애니메이션
         └─────────────┘
```

### 5.2 핵심 가치 상세

| 가치 | 설명 |
|------|------|
| **긴장감** | 매 턴 올라가는 확률, 심박 효과, 카운트다운으로 극대화 |
| **공정성** | 암호학적 수준의 RNG로 조작 불가능한 결과 보장 |
| **간편함** | URL 접속 → 이름 입력 → 플레이. 3단계 이내 게임 시작 |
| **재미** | 리볼버 테마, 총성/찰칵 사운드, 폭발 파티클, 화면 흔들림 |
| **사회성** | 결과 공유, 기록 저장으로 모임의 추억 생성 |

## 6. 경쟁 서비스 분석

| 서비스 | 방식 | 장점 | 단점 |
|--------|------|------|------|
| 사다리타기 앱 | 사다리 게임 | 익숙함, 심플 | 긴장감 부족, 연출 없음 |
| 제비뽑기 앱 | 랜덤 뽑기 | 빠른 결과 | 참여감 없음, 재미 없음 |
| 룰렛 돌리기 | 원형 룰렛 | 시각적 | 개인별 참여 부족 |
| 랜덤 숫자 생성 | 숫자 비교 | 초심플 | 연출 전무, 재미 없음 |

## 7. 차별화 전략

### 7.1 기존 서비스 대비 차별점

| 차별 요소 | 기존 서비스 | 러시안 룰렛 |
|-----------|-------------|-------------|
| 게임성 | 단순 결과 노출 | 턴제 참여, 확률 변화 |
| 긴장감 | 거의 없음 | 턴마다 상승, 심박/사운드 |
| 연출 | 간단한 애니메이션 | 탄창 회전, 총성, 파티클, 햅틱 |
| 참여도 | 결과만 확인 | 직접 방아쇠를 당기는 행위 |
| 재플레이 | 1회성 | 다양한 모드로 반복 플레이 |
| 모바일 | 앱 설치 필요 | 웹 접속 즉시 플레이 |

### 7.2 핵심 차별화 포인트
> **"직접 방아쇠를 당기는 행위"** 가 주는 심리적 긴장감은 다른 랜덤 도구에서 절대 제공할 수 없는 고유한 경험이다.

## 8. 서비스 운영 방식

### 8.1 배포 및 호스팅
- **GitHub Pages** 기반 정적 호스팅 (무료)
- 서버 불필요 — 모든 로직이 클라이언트에서 실행
- CDN 자동 적용으로 빠른 로딩

### 8.2 데이터 정책
- 서버에 어떤 데이터도 전송하지 않음
- 참가자 이름, 게임 기록은 브라우저 로컬 스토리지에만 저장
- 사용자가 원할 때 자유롭게 삭제 가능

### 8.3 업데이트
- GitHub 리포지토리 push → 자동 배포
- 버전별 릴리즈 태그 관리
