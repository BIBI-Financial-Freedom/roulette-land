## 2. F1 — 참가자 관리

### 2.1 기능 개요
게임에 참여할 참가자들의 이름을 등록·수정·삭제하는 기능.

### 2.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F1-1 | 참가자 이름 입력 필드 제공 | 필수 |
| F1-2 | 최소 2명, 최대 6명 제한 | 필수 |
| F1-3 | 참가자 동적 추가 (+ 버튼) | 필수 |
| F1-4 | 참가자 개별 삭제 (× 버튼) | 필수 |
| F1-5 | 빈 이름 입력 방지 (유효성 검사) | 필수 |
| F1-6 | 중복 이름 경고 | 권장 |
| F1-7 | 참가자 순서 드래그 정렬 | 선택 |
| F1-8 | 이전 참가자 목록 불러오기 (로컬 스토리지) | 선택 |

### 2.3 입력 제약

| 항목 | 제약 |
|------|------|
| 이름 길이 | 1~10자 |
| 허용 문자 | 한글, 영문, 숫자, 공백 |
| 참가자 수 | 최소 2, 최대 6 |

### 2.4 동작 흐름
```
[이름 입력 필드] → [+ 추가] → [이름 목록에 반영]
                                    │
                              [× 삭제] → [목록에서 제거]
                                    │
                              [2명 미만 시 시작 버튼 비활성화]
```

## 4. F3 — 탄창 시스템

### 4.1 기능 개요
6칸 리볼버 탄창의 생성, 탄환 배치, 스핀(셔플)을 담당하는 핵심 게임 엔진.

### 4.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F3-1 | 6칸 탄창 데이터 구조 생성 | 필수 |
| F3-2 | 탄환 무작위 배치 (RNG) | 필수 |
| F3-3 | 탄창 스핀 (시작 위치 무작위화) | 필수 |
| F3-4 | 모드별 탄환 수 적용 | 필수 |
| F3-5 | 페어 모드: 매 턴 리스핀 로직 | 필수 |
| F3-6 | 탄환 위치 보안 (클라이언트 노출 방지) | 필수 |

### 4.3 탄창 데이터 구조

```javascript
// Chamber 배열: 6칸, true = 탄환, false = 빈칸
const chamber = [false, false, true, false, false, false];

// 현재 위치 인덱스
let currentPosition = 0;

// 스핀: 시작 위치를 랜덤으로 설정
function spin() {
  currentPosition = getSecureRandom(0, 5);
}

// 방아쇠: 현재 위치의 값을 반환하고 다음으로 이동
function pull() {
  const result = chamber[currentPosition];
  currentPosition = (currentPosition + 1) % 6;
  return result; // true = 꽝, false = 생존
}
```

### 4.4 RNG 요구사항
- `crypto.getRandomValues()` 사용 (암호학적 안전 난수)
- `Math.random()` 사용 금지 (예측 가능성)
- 탄환 위치가 DOM이나 콘솔에서 노출되지 않도록 클로저로 캡슐화

### 5.1 기능 개요
참가자별 순서대로 턴을 진행하고, 방아쇠 입력을 처리하는 기능.

### 5.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F4-1 | 현재 차례 참가자 표시 | 필수 |
| F4-2 | 방아쇠 버튼 클릭 이벤트 | 필수 |
| F4-3 | 턴 순서 자동 전환 | 필수 |
| F4-4 | 순환 구조 (마지막 → 첫 번째) | 필수 |
| F4-5 | 꽝 발생 시 즉시 턴 종료 | 필수 |
| F4-6 | 카운트다운 타이머 (선택적) | 선택 |
| F4-7 | 타이머 만료 시 자동 발사 | 선택 |

### 5.3 턴 진행 상태 머신

```
[WAITING]          참가자 턴 대기, 방아쇠 버튼 활성화
    │
    ▼ (버튼 클릭)
[PULLING]          방아쇠 당기는 연출 재생 (0.5~1초)
    │
    ├── 빈 칸 ──▶ [SURVIVE]   생존 연출 → 다음 참가자 [WAITING]
    │
    └── 탄환 ──▶ [BANG]       꽝 연출 → [GAME_OVER]
```

### 5.4 상태 정의

| 상태 | 설명 | 전환 조건 |
|------|------|-----------|
| `WAITING` | 방아쇠 대기 | 버튼 클릭 또는 타이머 만료 |
| `PULLING` | 방아쇠 당기는 중 (연출) | 연출 완료 |
| `SURVIVE` | 생존 결과 연출 | 연출 완료 → 다음 턴 |
| `BANG` | 꽝 결과 연출 | 연출 완료 → GAME_OVER |
| `GAME_OVER` | 게임 종료 | 다시 하기 |

| F5-3 | 꽝 시 패배자 확정 | 필수 |
| F5-4 | 게임 종료 조건 판단 | 필수 |
| F5-5 | 서바이벌 모드: 탈락 처리 & 다음 라운드 | 선택 (v2.0) |
| F5-6 | 잔여 확률 실시간 계산 | 권장 |

### 6.3 판정 로직

```javascript
function judge(pullResult, gameMode) {
  if (pullResult === true) {
    // 꽝!
    return {
      type: 'BANG',
      loser: currentPlayer,
      penalty: selectedPenalty
    };
  } else {
    // 생존
    return {
      type: 'SURVIVE',
      survivor: currentPlayer,
      remainingChance: calculateRemaining()
    };
  }
}

function calculateRemaining() {
  const remaining = totalChambers - currentPosition;
  const bullets = chamber.filter(c => c === true).length;
  return `${bullets}/${remaining}`;
}
```
