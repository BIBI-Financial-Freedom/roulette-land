## 1. 프로젝트 개요

러시안 룰렛 방식으로 **꽝을 뽑는 사람을 결정하는 파티/벌칙 게임**이다.  
참가자들이 돌아가며 방아쇠를 당기고, 꽝(탄환)에 걸린 사람이 벌칙을 수행한다.

### 1.1 목적
- 모임에서 벌칙 대상을 스릴 넘치고 공정하게 결정
- 러시안 룰렛 특유의 긴장감과 서스펜스 연출로 재미 극대화
- 커피내기, 점심내기, 벌칙 수행자 등 다양한 상황에 활용 가능

### 1.2 대상 사용자
- 직장 동료, 친구, 스터디 그룹 등 2인 이상의 참가자

|------|------|

| 테마 | 리볼버 탄창(6칸) 기반의 확률 게임 |

## 3. 용어 정의

| 용어 | 정의 |
|------|------|
| **탄창 (Chamber)** | 리볼버의 회전 탄창. 총 6칸으로 구성되며 그 중 일부에 꽝(탄환)이 배치된다. |
| **탄환 (Bullet)** | 탄창에 숨겨진 꽝. 이 칸에 걸리면 패배한다. |
| **빈 칸 (Empty)** | 탄환이 없는 안전한 칸. 생존을 의미한다. |
| **방아쇠 (Trigger)** | 참가자가 자신의 턴에 누르는 버튼. 현재 칸의 결과를 공개한다. |
| **스핀 (Spin)** | 탄창을 무작위로 회전시켜 초기 위치를 섞는 동작. |
| **라운드 (Round)** | 꽝이 발생할 때까지 모든 참가자가 방아쇠를 당기는 한 사이클. |
| **생존 (Survive)** | 빈 칸에 걸려 탈락하지 않는 것. |
| **꽝 (BANG!)** | 탄환에 걸린 상태. 해당 참가자가 벌칙 대상이 된다. |
| **벌칙 (Penalty)** | 꽝에 걸린 참가자가 수행해야 하는 행동 (커피사기, 청소 등). |
| **RNG (Random Number Generator)** | 탄환 위치와 탄창 회전을 결정하는 난수 생성 로직. |

## 4. 게임 규칙

### 4.1 기본 규칙
1. **참가자 등록:** 2~6명의 참가자 이름을 입력한다.
2. **탄창 장전:** 6칸 탄창 중 1칸에 탄환(꽝)이 무작위 배치된다.
3. **탄창 스핀:** 게임 시작 시 탄창이 무작위로 회전하여 시작 위치를 결정한다.
4. **턴 진행:** 참가자가 순서대로 방아쇠 버튼을 누른다.
5. **결과 판정:**
   - **빈 칸** → 생존! 다음 참가자에게 턴이 넘어간다.
   - **탄환** → 꽝! 해당 참가자가 벌칙 대상으로 확정된다.
6. **게임 종료:** 꽝이 발생하면 즉시 라운드가 종료된다.

### 4.2 확률 변화
- 탄창은 한 번 스핀한 뒤 순차 진행되므로, 진행될수록 꽝 확률이 올라간다.
- 1번째: 1/6 (≈16.7%) → 2번째: 1/5 (20%) → 3번째: 1/4 (25%) → ... → 6번째: 1/1 (100%)
- 이 구조가 핵심 긴장감을 형성한다.

### 4.3 선택 규칙 (옵션)
- **매 턴 리스핀:** 매 참가자가 방아쇠를 당기기 전 탄창을 다시 돌려 확률을 항상 1/6으로 유지 (긴장감 감소, 공정성 증가)
- **멀티 탄환:** 탄환을 2~3개로 늘려 꽝 확률을 높임 (하드 모드)
- **벌칙 룰렛 연동:** 꽝에 걸린 사람이 수행할 벌칙을 별도 룰렛으로 결정

## 5. 탄창 구성

### 5.1 기본 모드 (Easy)

```
탄창 6칸: [ 💀 | ⬜ | ⬜ | ⬜ | ⬜ | ⬜ ]
탄환 수: 1개
꽝 확률: 1/6 (시작 시)
```

### 5.2 하드 모드

```
탄창 6칸: [ 💀 | 💀 | ⬜ | ⬜ | ⬜ | ⬜ ]
탄환 수: 2개
꽝 확률: 2/6 = 1/3 (시작 시)
```

### 5.3 데스 모드

```
탄창 6칸: [ 💀 | 💀 | 💀 | ⬜ | ⬜ | ⬜ ]
탄환 수: 3개
꽝 확률: 3/6 = 1/2 (시작 시)
```

## 6. 게임 모드

| 모드 | 설명 | 탄환 수 | 리스핀 |
|------|------|---------|--------|
| **클래식** | 순차 진행, 점점 확률 증가 | 1 | 없음 |
| **페어** | 매 턴 리스핀, 동일 확률 유지 | 1 | 매턴 |
| **하드** | 탄환 2개, 높은 꽝 확률 | 2 | 없음 |
| **데스매치** | 탄환 3개, 거의 반반 확률 | 3 | 없음 |
| **서바이벌** | 여러 라운드 진행, 최후 1인 생존 | 1 | 라운드별 |

### 6.1 서바이벌 모드 상세
- 매 라운드 꽝에 걸린 참가자가 탈락한다.
- 마지막 1명이 남을 때까지 반복한다.
- 최종 생존자는 면제, 탈락 순서에 따라 벌칙 등급이 결정된다.

## 7. 게임 흐름

```
[시작 화면]
     │
     ▼
[참가자 이름 입력] ── 2~6명
     │
     ▼
[게임 모드 선택]
     │
     ▼
[탄창 장전 연출] ── 탄환 배치 + 스핀 애니메이션
     │
     ▼
[라운드 시작]
     │
     ▼
[참가자 N의 턴]
     │
     ├── 방아쇠 클릭
     │      │
     │      ├── ⬜ 빈 칸 → "찰칵! 생존!" → 다음 참가자
     │      │
     │      └── 💀 탄환 → "빵! 꽝!" → 게임 종료
     │
     ▼ (꽝 발생)
[결과 발표]
  "💀 OOO님이 꽝에 걸렸습니다!"
  "벌칙: 커피 사기"
     │
     ▼
[다시 하기 / 종료]
```

## 8. 화면 구성

### 8.1 시작 화면
- 게임 타이틀 로고 (총알/리볼버 모티브)
- 참가자 이름 입력 폼 (동적 추가/삭제, 최소 2명 ~ 최대 6명)
- 게임 모드 선택 드롭다운 또는 카드 UI
- 벌칙 항목 입력 (선택사항)
- "게임 시작" 버튼

### 8.2 게임 화면
- **중앙:** 리볼버 탄창 UI (6칸 회전 원형)
  - 현재 칸 하이라이트
  - 이미 확인된 빈 칸 표시
  - 탄환 위치는 숨김 처리
- **상단:** 현재 차례 참가자 이름 + 프로필
- **하단:** 방아쇠 버튼 (큰 원형 버튼, 터치 친화적)
- **사이드:** 참가자 목록 + 생존/탈락 상태

### 8.3 결과 화면 — 생존 (빈 칸)
- "찰칵!" 텍스트 + 안도 애니메이션
- 탄창에서 해당 칸이 빈 칸으로 공개
- 자동으로 다음 참가자 턴 전환

### 8.4 결과 화면 — 꽝 (탄환)
- "💥 빵!" 폭발 이펙트 + 화면 흔들림
- 패배 참가자 강조 연출
- 벌칙 내용 대형 텍스트 표시
- "다시 하기" / "공유하기" 버튼

## 9. 연출 및 효과

### 9.1 사운드 효과
| 상황 | 효과음 |
|------|--------|
| 탄창 스핀 | 회전 기계음 (철컹철컹) |
| 방아쇠 당기기 전 | 심장박동 / 긴장 BGM |
| 빈 칸 (생존) | "찰칵" + 안도의 한숨 |
| 탄환 (꽝) | "빵!" 총성 + 폭발음 |
| 게임 종료 | 결과 팡파르 |

### 9.2 화면 연출
| 상황 | 연출 |
|------|------|
| 탄창 장전 | 탄환이 탄창에 들어가는 애니메이션 |
| 탄창 스핀 | 원형 탄창 고속 회전 → 감속 정지 |
| 방아쇠 대기 | 화면 미세 진동 + 심박 효과 |
| 생존 | 초록 플래시 + 체크마크 |
| 꽝 발생 | 빨간 플래시 + 화면 흔들림 + 폭발 파티클 |
| 패자 발표 | 스포트라이트 + 이름 확대 |

### 9.3 긴장감 연출
- 남은 빈 칸이 줄어들수록 심박 효과 빨라짐
- 방아쇠 버튼에 카운트다운 제한 시간 옵션 (망설이면 자동 발사)
- 탄창 잔여 칸 수 시각적 표시 (잔여 확률 계산)
