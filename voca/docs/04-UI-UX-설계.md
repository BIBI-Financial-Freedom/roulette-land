## 1. 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **도파민 폭발** | 형광 네온 컬러 + 과장된 이펙트로 중독성 있는 시각 경험 |
| **다크 배경 × 형광 강조** | 어두운 배경 위에 형광 시안/옐로우/핑크가 터지는 느낌 |
| **한 화면 한 액션** | 닉네임 → 주제 → 입력 → 로딩 → 결과, 단계별 전환 |
| **모바일 퍼스트** | 90%가 카카오톡 링크 클릭 유입. 세로형 레이아웃 기본 |
| **직관적 버튼** | 큰 크기, 고대비, 명확한 레이블. 터치 최적화 |
| **텍스트 입력 편의** | textarea 큰 크기, 글자 수 카운터, 자동 포커스 |

## 6. 인터랙션 설계

### 6.1 화면 전환

| 전환 | 애니메이션 | 시간 |
|------|-----------|------|
| Step1 → Step2 | 좌로 슬라이드 | 0.4s |
| Step2 → 로딩 | 페이드 아웃 + 줌 인 | 0.4s |
| 로딩 → 결과 | 글리치 + 페이드 인 | 0.6s |
| 결과 → Step1 (다시하기) | 페이드 + 스케일 다운 | 0.3s |

### 6.2 주제 카드 선택

| 액션 | 효과 |
|------|------|
| 터치/클릭 | 테두리 시안 글로우 + scale(1.02) |
| 선택 해제 (다른 카드 클릭) | 이전 카드 글로우 페이드아웃 |

### 6.3 글자 수 카운터

| 상태 | 표시 |
|------|------|
| 0~29자 | "❌ 15자 / 최소 30자" (빨간) |
| 30~499자 | "✅ 98자 / 최소 30자" (초록) |
| 500자 | "⚠️ 500자 / 최대 500자" (노랑) |

### 6.4 결과 화면 연출

| 요소 | 연출 |
|------|------|
| 어휘 연령 숫자 | 0에서 카운트업 → 목표값 도달 + 마젠타 글로우 펄스 |
| 문해력 게이지 | 0% → 목표값 좌→우 채움 |
| 반복 단어 카드 | 좌→우 순차 팝업 (0.2s 간격) |
| 팩폭 코멘트 | 타이핑 애니메이션 |
| 전체 카드 | 약한 떨림 후 안정 |

## 8. 반응형 설계

### 8.1 브레이크포인트

| 디바이스 | 범위 | 최적화 |
|----------|------|--------|
| 모바일 (기본) | ~480px | 풀스크린, 세로 스크롤 |
| 태블릿 | 481~768px | 중앙 정렬, 최대 너비 480px |
| 데스크탑 | 769px~ | 중앙 정렬, 최대 너비 420px, 모바일 프레임 |

### 8.2 모바일 최적화 사항

| 항목 | 설정 |
|------|------|
| 최소 터치 영역 | 48×48px |
| textarea 폰트 | 16px (iOS 자동 줌 방지) |
| 키보드 대응 | 연 시 레이아웃 조정 |
| 스크롤 | `scroll-behavior: smooth` |
| safe-area | `env(safe-area-inset-*)` 적용 |
| 주제 카드 그리드 | 2열 그리드 (모바일), 4열 (데스크탑) |

## 9. 접근성

| 항목 | 대응 |
|------|------|
| 색상 대비 | WCAG AA 기준 충족 (네온 컬러 대비비 4.5:1+) |
| 키보드 탐색 | Tab 순서 보장, 주제 카드 키보드 선택 |
| 스크린 리더 | aria-label, role 적용 |
| textarea | label 연결, 남은 글자 수 aria-live |

## 10. 사용자 흐름 (User Flow)

```
[서비스 접속]
     │
     ├─ [URL 파라미터 있음?]
     │     │
     │     ├─ YES: OG 태그 결과 반영 → 입력 화면으로
     │     └─ NO: 기본 OG 태그
     │
     ▼
[Step 1: 닉네임 + 주제 선택]
     │
     ├─ 닉네임 입력
     ├─ 주제 카드 1개 선택
     │
     ▼
[Step 2: 텍스트 입력]
     │
     ├─ textarea에 3문장+ 입력
     ├─ 글자 수 30자 이상 확인
     │
     ▼
[어휘력 분석하기 버튼]
     │
     ▼
[로딩 화면]
     │  사전 애니메이션
     │  분석 멘트 표시
     │  프로그레스 바
     │
     ▼
[결과 화면]
     │
     ├─ [이미지 저장] → PNG 다운로드 (1080×1920)
     │
     ├─ [공유하기] → OG 파라미터 포함 URL 복사
     │
     └─ [다시 해보기] → [Step 1]으로 돌아감
```
