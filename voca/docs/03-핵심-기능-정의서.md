## 4. F3 — 텍스트 입력

### 4.1 기능 개요
유저가 선택한 주제에 맞는 텍스트를 3문장 이상 자유롭게 입력하는 기능.

### 4.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F3-1 | textarea 입력 필드 제공 | 필수 |
| F3-2 | 최소 30자 이상 입력 필수 | 필수 |
| F3-3 | 최대 500자 제한 | 필수 |
| F3-4 | 실시간 글자 수 카운터 | 필수 |
| F3-5 | 30자 미만 시 제출 버튼 비활성화 | 필수 |
| F3-6 | 주제별 플레이스홀더 다르게 | 권장 |
| F3-7 | XSS 방지 새니타이징 | 필수 |

### 4.3 플레이스홀더 예시

| 주제 | 플레이스홀더 |
|------|-------------|
| 변명 | "아 진짜 이거 제 잘못이 아니라..." |
| 돈 빌리기 | "야 나 진짜 급한데 좀만 빌려줘 ㅠㅠ..." |
| 이별 통보 | "우리 좀 얘기 좀 하자..." |
| 지각 변명 | "미안 지금 거의 다 왔어..." |

### 4.4 동작 흐름

```
[주제 선택 완료]
     │
     ▼
[textarea 활성화]
     │
     ├─ 입력 중 → 글자 수 실시간 표시
     │              "현재 45자 / 최소 30자"
     │
     ├─ 30자 미만 → 제출 버튼 비활성화 (회색)
     │
     └─ 30자 이상 → 제출 버튼 활성화 (네온 글로우)
                          │
                          ▼
                    [분석하기 버튼]
```

## 5. F4 — AI 어휘 분석 엔진

### 5.1 기능 개요
LLM API를 호출하여 유저 텍스트의 어휘력을 분석하고, 어휘 연령 + 팩폭 코멘트를 생성하는 핵심 엔진.

### 5.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F4-1 | LLM API 호출 (OpenAI / Gemini) | 필수 |
| F4-2 | 프롬프트에 닉네임 + 주제 + 텍스트 포함 | 필수 |
| F4-3 | 어휘 연령 (세) 생성 | 필수 |
| F4-4 | 문해력 점수 (0~100) 생성 | 필수 |
| F4-5 | 반복 단어 Top 3 추출 | 필수 |
| F4-6 | 어휘 분석 팩폭 코멘트 생성 | 필수 |
| F4-7 | 한 줄 요약 (OG/공유용) 생성 | 필수 |
| F4-8 | API 에러 시 폴백 메시지 | 필수 |
| F4-9 | API 호출 타임아웃 (15초) | 필수 |
| F4-10 | API Key 보안 (프록시 경유) | 필수 |

### 5.3 프롬프트 구조 (예시)

```
너는 날카로운 국어 선생님이야, 학생의 어휘력을 날카롭게 분석하고 조롱해.

유저 닉네임: {nickname}
주제: {topic}
유저가 쓴 텍스트:
"""
{userText}
"""

다음 JSON 형식으로 응답해:
{
  "vocabAge": 7,           // 정신적 어휘 연령 (5~80세)
  "score": 23,             // 문해력 점수 (0~100)
  "repeatedWords": [       // 가장 많이 반복한 단어 Top 3
    {"word": "진짜", "count": 5},
    {"word": "완전", "count": 3},
    {"word": "대박", "count": 2}
  ],
  "analysis": "...",       // 어휘 분석 팩폭 코멘트 (3~4문장)
  "roast": "...",          // 한 줄 핵심 디스 (OG 태그용)
  "keyword": "...",        // 어휘 특성 키워드 1개 (예: "복사붙여넣기급")
  "era": "2010년대"        // 어휘가 머물러 있는 시대
}
```

### 5.4 응답 파싱

```javascript
// LLM 응답 구조
{
  "vocabAge": 7,
  "score": 23,
  "repeatedWords": [
    { "word": "진짜", "count": 5 },
    { "word": "완전", "count": 3 },
    { "word": "대박", "count": 2 }
  ],
  "analysis": "님 어휘력은 2010년대에 머물러 있네요. '진짜', '완전', '대박' 삼총사 없이는 한 문장도 완성을 못 하시네요. 이건 어휘력이 아니라 복사-붙여넣기 수준입니다. 3문장 중 독창적인 표현이 0개입니다.",
  "roast": "'진짜'를 빼면 말을 못 하는 7세 수준",
  "keyword": "복사붙여넣기급",
  "era": "2010년대"
}
```

### 5.5 폴백 처리

| 상황 | 대응 |
|------|------|
| API 응답 지연 (15초+) | 타임아웃 메시지 + 재시도 버튼 |
| API 에러 (4xx/5xx) | 사전 준비된 랜덤 분석 결과 사용 |
| JSON 파싱 실패 | 원본 텍스트에서 간단한 통계 추출 (클라이언트) |
| Rate Limit | "문해력 테스트 인기 폭발! 잠시 후 다시 시도해 주세요" |

## 9. F8 — OG 어그로 공유

### 9.1 기능 개요
결과를 URL 파라미터에 인코딩하여 카카오톡 미리보기에 결과가 노출되도록 하는 핵심 바이럴 기능.

### 9.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F8-1 | 결과 URL 생성 (파라미터 포함) | 필수 |
| F8-2 | URL 복사 (Clipboard API) | 필수 |
| F8-3 | 복사 완료 토스트 메시지 | 필수 |
| F8-4 | OG meta 태그 동적 업데이트 | 필수 |
| F8-5 | Web Share API 지원 (모바일) | 권장 |
| F8-6 | URL 파라미터 XSS 방지 | 필수 |

### 9.3 URL 파라미터 설계

```
// 기본 URL
https://example.com/voca/

// 결과 공유 URL
https://example.com/voca/?name={닉네임}&age={어휘연령}&keyword={키워드}

// 예시
https://example.com/voca/?name=이현민&age=7&keyword=복사붙여넣기급
```

### 9.4 OG 태그 동적 생성

```html
<!-- 기본 OG (결과 파라미터 없을 때) -->
<meta property="og:title" content="📖 문해력 바닥 인증기">
<meta property="og:description" content="당신의 어휘력은 몇 세 수준일까요?">

<!-- 결과 공유 OG (파라미터 있을 때) -->
<meta property="og:title" content="📖 문해력 바닥 인증기">
<meta property="og:description" content="이현민 님의 어휘력은 7세 수준입니다">
```

### 9.5 OG 파라미터 처리 흐름

```
[URL에 파라미터 있음?]
     │
     ├─ YES: OG 태그에 결과 반영
     │        → 카카오톡 미리보기에 결과 노출
     │        → 클릭 시 일반 입력 화면으로 이동
     │
     └─ NO: 기본 OG 태그
              → "당신의 어휘력은 몇 세 수준일까요?"
```
