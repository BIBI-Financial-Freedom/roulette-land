## 1. 서비스 소개

**Operation Epic Fury**는 팀·친구 간 커피내기를 슬롯머신 게임으로 결정하는 웹 기반 캐주얼 파티 게임이다.

참가자들이 각자 순서대로 슬롯을 돌려 점수를 경쟁하고, 최하위 점수자가 커피를 사는 구조로 — 단순한 제비뽑기를 넘어 **긴장감·재미·공정성**을 모두 확보한다.

---

## 2. 서비스 비전

| 항목 | 내용 |
|------|------|
| **비전** | "커피 한 잔의 내기를 인생 최고의 긴장감으로" |
| **미션** | 누구나 3초 안에 시작할 수 있는 초간단 내기 게임 제공 |
| **핵심 가치** | 공정성 · 재미 · 접근성 · 심플함 |

---

## 3. 서비스 포지셔닝

```
                재미 ↑
                     │
        ┌────────────┼────────────────┐
        │            │    ★ Epic Fury │
        │            │   (슬롯+연출)   │
        │            │                │
  단순 ─┼────────────┼────────────────┼─ 기능 풍부
        │ 사다리타기  │                │
        │ 제비뽑기    │   랜덤 룰렛    │
        │            │                │
        └────────────┼────────────────┘
                     │
                재미 ↓
```

**차별점:** 기존 랜덤 도구(사다리타기, 제비뽑기)는 결과만 보여주지만, Epic Fury는 **슬롯머신 연출 + 점수 시스템 + 벌칙 엔터테인먼트**로 과정 자체가 재미있다.

## 4. 타겟 사용자

### 4.1 주요 타겟 (Primary)

| 페르소나 | 설명 |
|----------|------|
| **직장인 김 대리** | 점심 후 팀원 4명과 커피 내기를 매일 함. 매번 가위바위보는 지루함. |
| **대학생 이 모씨** | 스터디 모임에서 간식 내기를 자주 함. 모바일로 바로 쓸 수 있는 게 필요함. |

### 4.2 보조 타겟 (Secondary)

| 페르소나 | 설명 |
|----------|------|
| **모임장 박 과장** | 회식/워크숍에서 벌칙게임 요소로 활용. |
| **커플 최 모씨** | 데이트 중 메뉴 결정, 계산 내기에 활용. |

## 5. 핵심 사용자 시나리오

### 시나리오 1: 점심 후 커피 내기
```
1. 김 대리가 브라우저에서 게임 접속 (URL 공유 or 즐겨찾기)
2. 팀원 4명 이름 입력: 김대리, 이사원, 박주임, 최인턴
3. "게임 시작" 클릭
4. 각자 순서대로 스핀 → 릴 회전 → 결과 확인
5. 최하위: 최인턴 (올 디퍼런트, 100점)
6. 화면에 "☕ 최인턴님이 커피를 삽니다!" 연출
7. 모두 웃으며 커피 주문
```

### 시나리오 2: 서든데스 상황
```
1. 3명 참가, 스핀 완료
2. 이사원(100점)과 박주임(100점) 동률 최하위
3. 서든데스 진입 → 두 명만 추가 스핀
4. 이사원(200점) vs 박주임(150점) → 박주임 패배
5. "☕ 박주임님이 커피를 삽니다!"
```

## 8. 서비스 운영 정책

### 8.1 가용성
- 정적 호스팅 기반으로 **서버 다운타임 없음**
- CDN 캐시를 통한 빠른 로딩

### 8.2 브라우저 호환성

| 브라우저 | 최소 버전 |
|----------|----------|
| Chrome | 90+ |
| Safari | 14+ |
| Firefox | 88+ |
| Edge | 90+ |
| 모바일 Chrome/Safari | 최신 2개 버전 |

### 8.3 성능 목표

| 지표 | 목표값 |
|------|--------|
| 첫 로딩 (FCP) | < 1.5초 |
| 인터랙션 준비 (TTI) | < 2초 |
| 번들 사이즈 | < 500KB (gzipped) |
| 릴 애니메이션 FPS | 60fps |

## 9. 성공 지표 (KPI)

비상업 프로젝트이므로 사용자 만족 중심의 정성적 지표를 활용한다.

| 지표 | 측정 방법 | 목표 |
|------|----------|------|
| 접근성 | URL 접속 → 첫 스핀까지 소요 시간 | 30초 이내 |
| 재사용 | 같은 사용자 재방문 여부 (로컬 스토리지 기반) | 주 2회 이상 |
| 완주율 | 게임 시작 → 결과 화면 도달 비율 | 95% 이상 |
| 공유율 | 결과 공유 버튼 클릭 비율 | 30% 이상 |
