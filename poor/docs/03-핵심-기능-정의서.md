## 2. F1 — 닉네임 입력

### 2.1 기능 개요
결과에 반영할 유저 닉네임을 입력받는 기능. 개인화된 팩폭의 핵심.

### 2.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F1-1 | 닉네임 입력 필드 제공 | 필수 |
| F1-2 | 1~10자 제한 | 필수 |
| F1-3 | 빈 입력 방지 (유효성 검사) | 필수 |
| F1-4 | 한글, 영문, 숫자 허용 | 필수 |
| F1-5 | 플레이스홀더: "파산 예정자 이름" | 권장 |

### 2.3 입력 제약

| 항목 | 제약 |
|------|------|
| 길이 | 1~10자 |
| 허용 문자 | 한글, 영문, 숫자, 공백 |
| XSS 방지 | 특수문자 이스케이프 처리 |

## 3. F2 — 소비 고백 입력

### 3.1 기능 개요
유저가 최근 가장 쓸데없이 돈 쓴 곳 3가지를 자유 텍스트로 입력하는 기능.

### 3.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F2-1 | 소비 입력 필드 3개 제공 | 필수 |
| F2-2 | 각 필드 1~50자 제한 | 필수 |
| F2-3 | 3개 모두 입력해야 제출 가능 | 필수 |
| F2-4 | 빈 입력 방지 | 필수 |
| F2-5 | 플레이스홀더 예시 제공 | 권장 |
| F2-6 | 입력 내용 XSS 필터링 | 필수 |

### 3.3 플레이스홀더 예시

| 필드 | 플레이스홀더 |
|------|-------------|
| 소비 1 | "예: 탕후루 3만 원어치" |
| 소비 2 | "예: 안 입는 요가복" |
| 소비 3 | "예: 쓸데없는 구독 서비스" |

### 3.4 동작 흐름

```
[닉네임 입력 완료]
     │
     ▼
[소비 1 입력] → [소비 2 입력] → [소비 3 입력]
                                      │
                                      ▼
                              [유효성 검사]
                                      │
                         ┌────────────┴────────────┐
                         │                         │
                   [통과 → 제출]            [실패 → 에러 표시]
                         │
                         ▼
                   [AI 분석 시작]
```

## 4. F3 — AI 팩폭 엔진

### 4.1 기능 개요
LLM API를 호출하여 유저의 소비 내역을 기반으로 파산 예언 텍스트를 생성하는 핵심 엔진.

### 4.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F3-1 | LLM API 호출 (OpenAI / Gemini) | 필수 |
| F3-2 | 프롬프트에 닉네임 + 소비 3개 포함 | 필수 |
| F3-3 | 파산 예정 시점 (년/월) 생성 | 필수 |
| F3-4 | 파산 후 모습 묘사 (유머러스) | 필수 |
| F3-5 | 거지력 수치 (0~100%) 생성 | 필수 |
| F3-6 | 각 소비별 개별 팩폭 코멘트 | 권장 |
| F3-7 | API 에러 시 폴백 메시지 | 필수 |
| F3-8 | API 호출 타임아웃 (15초) | 필수 |
| F3-9 | API Key 보안 (환경변수 / 프록시) | 필수 |

### 4.3 프롬프트 구조 (예시)

```
너는 독설가 금융 전문가야. 
유저의 쓸데없는 소비 습관을 보고 파산 시점을 예언하고 조롱해줘.

유저 닉네임: {nickname}
쓸데없는 소비 1: {expense1}
쓸데없는 소비 2: {expense2}
쓸데없는 소비 3: {expense3}

다음 형식으로 응답해줘:
1. 파산 예정일 (YYYY년 MM월)
2. 거지력 수치 (0~100%)
3. 파산 후 모습 묘사 (2~3문장, 구체적이고 웃긴 상황)
4. 각 소비에 대한 팩폭 코멘트 (각 1문장)
5. 한 줄 요약 (SNS 공유용)
```

### 4.4 응답 파싱

```javascript
// LLM 응답 JSON 구조
{
  "bankruptDate": "2026년 11월",
  "poorScore": 98,
  "futureDescription": "한강 대교 밑에서 무료 와이파이를 구걸하는 ...",
  "roasts": [
    "탕후루에 3만 원이요? 그 탕후루가 녹는 속도보다...",
    "안 입는 요가복? 요가보다 통장 잔고 스트레칭이...",
    "구독 서비스? 정작 구독해야 할 건 절약 습관인데..."
  ],
  "summary": "이현민 님은 2026년 11월 사이버 각설이 확정"
}
```

### 4.5 폴백 처리

| 상황 | 대응 |
|------|------|
| API 응답 지연 (15초+) | 타임아웃 메시지 + 재시도 버튼 |
| API 에러 (4xx/5xx) | 사전 준비된 랜덤 팩폭 메시지 사용 |
| JSON 파싱 실패 | 원본 텍스트 그대로 표시 |
| Rate Limit | "인기 폭발! 잠시 후 다시 시도해 주세요" |

## 6. F5 — 결과 카드 생성

### 6.1 기능 개요
결과 영역을 1080×1920 비율의 이미지로 렌더링하는 기능.

### 6.2 상세 요구사항

| ID | 요구사항 | 필수 여부 |
|----|----------|-----------|
| F5-1 | html2canvas로 결과 영역 캡처 | 필수 |
| F5-2 | 출력 해상도: 1080×1920px (9:16 비율) | 필수 |
| F5-3 | 결과 카드 전용 스타일링 (캡처 최적화) | 필수 |
| F5-4 | 서비스 워터마크/로고 삽입 | 권장 |
| F5-5 | 배경 그라데이션 포함 | 권장 |

### 6.3 캡처 영역 설정

```javascript
// 캡처 대상 DOM 요소
const captureTarget = document.getElementById('result-card');

// html2canvas 옵션
const options = {
  width: 1080,
  height: 1920,
  scale: 2,           // 고해상도
  backgroundColor: null,
  useCORS: true
};
```
