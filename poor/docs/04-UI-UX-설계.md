## 1. 디자인 원칙

| 원칙 | 설명 |
|------|------|
| **도파민 폭발** | 형광 네온 컬러 + 과장된 이펙트로 자극적인 시각 경험 |
| **다크 배경 × 형광 강조** | 어두운 배경 위에 형광 그린/핑크/옐로우가 터지는 느낌 |
| **한 화면 한 액션** | 입력 → 로딩 → 결과, 각 단계별 풀스크린 전환 |
| **모바일 퍼스트** | 90%가 모바일 유입. 세로형 레이아웃 기본 |
| **직관적 버튼** | 큰 크기, 고대비, 명확한 레이블. 손가락으로 쉽게 터치 |
| **즉각적 피드백** | 모든 인터랙션에 시각·촉각(햅틱) 반응 |

## 6. 인터랙션 설계

### 6.1 화면 전환

| 전환 | 애니메이션 | 시간 |
|------|-----------|------|
| 입력 → 로딩 | 페이드 아웃 + 슬라이드 업 | 0.4s |
| 로딩 → 결과 | 글리치 이펙트 + 페이드 인 | 0.6s |
| 결과 → 입력 (다시하기) | 페이드 + 스케일 다운 | 0.3s |

### 6.2 버튼 인터랙션

| 액션 | 효과 |
|------|------|
| 터치/클릭 | scale(0.95) + 네온 글로우 펄스 |
| 호버 | scale(1.05) + 글로우 증가 |
| 제출 | 버튼 → 로딩 스피너 변환 |
| 다운로드 완료 | 체크마크 ✅ 표시 + 진동 (햅틱) |

### 6.3 로딩 연출

| 단계 | 연출 | 시간 |
|------|------|------|
| 1단계 | 프로그레스 바 시작 (0%→30%) | 0~1s |
| 2단계 | 통장 잔고 급감 카운터 | 1~3s |
| 3단계 | 팩폭 멘트 순차 표시 | 3~5s |
| 4단계 | 프로그레스 바 완료 (100%) | API 응답 시 |
| 5단계 | 글리치 이펙트 → 결과 화면 전환 | 0.6s |

### 6.4 결과 화면 연출

| 요소 | 연출 |
|------|------|
| 파산 예정일 | 타이핑 애니메이션 + 네온 핑크 글로우 |
| 거지력 게이지 | 0% → 목표값 까지 카운트업 | 
| 팩폭 코멘트 | 순차 페이드인 (0.3s 간격) |
| 전체 카드 | 약한 떨림(shake) 효과 후 안정 |

## 8. 반응형 설계
