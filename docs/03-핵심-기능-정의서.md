## 2. F1 — 참가자 관리

### F1.1 참가자 등록

| 항목 | 상세 |
|------|------|
| **기능 ID** | F1.1 |
| **우선순위** | P0 (필수) |
| **설명** | 게임에 참여할 사용자의 이름을 등록한다 |

**기능 요구사항:**
- 이름 입력 필드 + "추가" 버튼으로 참가자를 동적으로 추가
- 최소 2명, 최대 10명까지 등록 가능
- 빈 이름 또는 공백만으로 구성된 이름은 등록 불가
- 중복 이름 등록 시 경고 표시 (동명이인은 번호 붙여 허용: "김철수", "김철수2")
- 등록된 참가자는 리스트로 표시

**입력:**
- `string` : 참가자 이름 (1~20자, 공백 제외 1자 이상)

**출력:**
- 참가자 리스트 UI 업데이트

**검증 규칙:**
```
이름.trim().length >= 1
이름.trim().length <= 20
참가자수 >= 2 && 참가자수 <= 10
```

## 3. F2 — 슬롯 엔진

### F2.1 릴 시스템

| 항목 | 상세 |
|------|------|
| **기능 ID** | F2.1 |
| **우선순위** | P0 (필수) |
| **설명** | 3개 릴을 독립적으로 회전시키고 결과 심볼을 표시한다 |

**기능 요구사항:**
- 3개 릴이 좌→우 순서로 약간의 시차를 두고 정지
- 각 릴의 정지 시 심볼은 RNG(F2.2)에 의해 결정
- 릴 회전 시 세로 스크롤 애니메이션 표현
- 정지 시점:
  - 릴 1: 스핀 후 1.0초
  - 릴 2: 스핀 후 1.5초
  - 릴 3: 스핀 후 2.0초
- 정지 타이밍은 ±0.3초 랜덤 변동

**릴 데이터 구조:**
```typescript
interface Reel {
  id: number;           // 0, 1, 2
  symbols: Symbol[];    // 가용 심볼 풀
  currentSymbol: Symbol; // 현재 표시 심볼
  isSpinning: boolean;
  stopDelay: number;    // ms
}

interface Symbol {
  id: string;
  emoji: string;        // ☕, 🫘, 🍩, 🥛, 🧊, 💧
  grade: 'S' | 'A' | 'B' | 'C' | 'D';
  weight: number;       // 가중치 (확률)
}
```

### F2.2 RNG (난수 생성)

| 항목 | 상세 |
|------|------|
| **기능 ID** | F2.2 |
| **우선순위** | P0 (필수) |
| **설명** | 각 릴의 결과 심볼을 가중치 기반으로 무작위 결정한다 |

**기능 요구사항:**
- `crypto.getRandomValues()` 기반의 암호학적 안전한 난수 사용
- 각 릴은 독립적으로 심볼을 결정 (릴 간 상관관계 없음)
- 가중치 확률 분포를 준수

**알고리즘 (가중치 기반 선택):**
```
weightedRandom(symbols):
  totalWeight = sum(symbol.weight for symbol in symbols)
  random = cryptoRandom(0, totalWeight)
  cumulative = 0
  for symbol in symbols:
    cumulative += symbol.weight
    if random < cumulative:
      return symbol
```

**심볼별 가중치:**
| 심볼 | 가중치 | 단일 릴 출현 확률 |
|------|--------|------------------|
| ☕ 에스프레소 | 5 | 5% |
| 🫘 원두 | 15 | 15% |
| 🍩 도넛 | 20 | 20% |
| 🥛 우유 | 20 | 20% |
| 🧊 각설탕 | 20 | 20% |
| 💧 물 | 20 | 20% |
| **합계** | **100** | **100%** |

**조합별 이론 확률:**
| 조합 | 계산 | 확률 |
|------|------|------|
| ☕☕☕ (잭팟) | 0.05³ | 0.0125% |
| 💧💧💧 (즉시 패배) | 0.20³ | 0.8% |
| 임의 트리플 매치 | Σ(p³) | ~3.55% |
| 더블 매치 (아무거나) | 6×C(3,2)×pᵢ²×(1-pᵢ) | ~26.1% |

## 4. F3 — 스코어링 시스템

### F3.1 점수 산출

| 항목 | 상세 |
|------|------|
| **기능 ID** | F3.1 |
| **우선순위** | P0 (필수) |
| **설명** | 스핀 결과의 심볼 조합을 분석하여 점수를 부여한다 |

**점수 테이블:**
| 조합 유형 | 조건 | 점수 |
|----------|------|------|
| 트리플 에스프레소 | ☕☕☕ | 1000 |
| 트리플 매치 (기타) | 동일 심볼 3개 (☕ 제외) | 500 |
| 더블 에스프레소 | ☕ 2개 + 다른 심볼 | 300 |
| 더블 매치 | 동일 심볼 2개 + 다른 심볼 | 200 |
| 에스프레소 싱글 | ☕ 1개 + 서로 다른 심볼 2개 | 150 |
| 올 디퍼런트 | 3개 모두 다름 (☕ 없음) | 100 |
| 트리플 물 | 💧💧💧 | 0 (즉시 패배) |

**판정 알고리즘:**
```
calculateScore(symbols[3]):
  // 즉시 패배 체크
  if all symbols == 💧: return { score: 0, instant_lose: true }
  
  // 트리플 매치
  if all symbols are same:
    if symbol == ☕: return { score: 1000, jackpot: true }
    else: return { score: 500 }
  
  // 더블 매치
  counts = countEachSymbol(symbols)
  if any count == 2:
    if doubled symbol == ☕: return { score: 300 }
    else: return { score: 200 }
  
  // 싱글 체크
  if ☕ in symbols: return { score: 150 }
  
  // 올 디퍼런트
  return { score: 100 }
```

### F3.3 서든데스

| 항목 | 상세 |
|------|------|
| **기능 ID** | F3.3 |
| **우선순위** | P0 (필수) |
| **설명** | 최하위 동점자 발생 시 추가 스핀으로 최종 패자를 결정한다 |

**기능 요구사항:**
- 최하위 점수가 동점인 참가자들만 추출
- 해당 참가자들끼리 추가 스핀 1회 진행
- 추가 스핀에서도 동점이면 반복 (최대 5회)
- 5회 반복 후에도 동점이면 해당 참가자들 **모두 패배** (공동 커피 구매)
- 서든데스 진입 시 특별 연출 (화면 효과 + 사운드)

**흐름:**
```
suddenDeath(tiedPlayers):
  for round in 1..5:
    각 동점자 스핀 진행
    새 점수로 최하위 판정
    if 최하위 1명만 존재: return loser
  return allTiedPlayers  // 공동 패배
```

## 5. F4 — 결과 및 연출

### F4.1 순위 집계

| 항목 | 상세 |
|------|------|
| **기능 ID** | F4.1 |
| **우선순위** | P0 (필수) |
| **설명** | 모든 참가자의 점수를 집계하여 순위를 매긴다 |

**기능 요구사항:**
- 높은 점수 = 높은 순위
- 즉시 패배(💧💧💧) 참가자는 무조건 최하위
- 동일 점수는 동일 순위 표시 (1위, 2위, 2위, 4위 방식)
- 순위 테이블 UI: 순위 / 이름 / 심볼 결과 / 점수

**데이터 구조:**
```typescript
interface RoundResult {
  rankings: PlayerResult[];
  loser: Player | Player[];  // 서든데스 공동 패배 가능
  hasJackpot: boolean;
  hasSuddenDeath: boolean;
}

interface PlayerResult {
  player: Player;
  spinResult: SpinResult;
  rank: number;
}
```

### F4.2 패자 연출

| 항목 | 상세 |
|------|------|
| **기능 ID** | F4.2 |
| **우선순위** | P0 (필수) |
| **설명** | 최종 패자(커피 셔틀)를 드라마틱하게 발표한다 |

**연출 시퀀스:**
```
1. 순위 테이블 순차 공개 (1위부터 차례로, 각 0.5초 간격)
2. 최하위 도달 시 잠시 멈춤 (1초 일시정지)
3. 드럼롤 사운드 재생
4. 화면 플래시 + 패자 이름 대형 표시
5. "☕ {이름}님이 커피를 삽니다!" 텍스트
6. 컨페티 또는 폭죽 이펙트 (패자 입장에서 약간의 위로 효과)
7. 잭팟 당첨자가 있으면 별도 축하 연출
```

**연출 요소:**
| 요소 | 구현 방식 |
|------|----------|
| 화면 플래시 | CSS `animation: flash` (배경색 전환) |
| 드럼롤 | 오디오 파일 재생 (Web Audio API) |
| 대형 텍스트 | CSS `scale` 애니메이션 (0 → 1) |
| 컨페티 | Canvas 파티클 또는 CSS 라이브러리 |
| 화면 흔들림 | CSS `transform: translate` 진동 |

## 6. F5 — 옵션 기능

### F5.1 리스핀

| 항목 | 상세 |
|------|------|
| **기능 ID** | F5.1 |
| **우선순위** | P2 (향후) |
| **설명** | 결과가 불만족스러울 때 1회 다시 돌릴 수 있다 |

**규칙:**
- 라운드당 참가자별 1회 사용 가능
- 리스핀 결과는 무조건 수용 (원래 결과로 되돌리기 불가)
- 리스핀 사용 시 "RESPIN!" 연출

### F5.2 더블 오어 낫싱

| 항목 | 상세 |
|------|------|
| **기능 ID** | F5.2 |
| **우선순위** | P2 (향후) |
| **설명** | 잭팟 당첨 시 2배 보상 또는 2배 벌칙을 건 도박 |

**규칙:**
- 잭팟(☕☕☕) 당첨자만 선택 가능
